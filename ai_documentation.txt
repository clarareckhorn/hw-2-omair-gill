Q: how to get ligotools folder to work as a proper Python package
A: Move the readligo.py file into a new folder called ligotools, then create an empty __init__.py file so Python recognizes it as a package.
mkdir ligotools
mv readligo.py ligotools/
touch ligotools/__init__.py

Q: How do I create a working Conda environment?
A: Create environment.yml file (nano environment.yml) that defines dependencies needed for the notebook to run
Paste this inside:

name: ligo
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.10
  - numpy
  - scipy
  - matplotlib
  - h5py
  - pandas
  - ipykernel
  - ipython
  - decorator=5.1
  - jupyterlab

Q: How to fix Binder errors (blackman/tukey)?
A: One option: 
signal.blackman(...) → signal.windows.blackman(...)
signal.tukey(...) → signal.windows.tukey(...)

Q: How do I rename the ligotools-pkg-3 to just ligotools-pkg
A: git push origin --delete ligotools-pkg-3
git tag -d ligotools-pkg-3
git tag ligotools-pkg
git push origin ligotools-pkg
